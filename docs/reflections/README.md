# Session Reflections

> Post-mission retrospectives that capture what worked, what didn't, and what to improve.
> Agents should read the 2–3 most recent reflections before starting large new changes.

## File Naming Convention

```
YYYY-MM-DD-<slug>.md
```

**Examples:**
- `2026-03-11-auth-feature.md`
- `2026-03-15-ui-overhaul.md`

## Template

```markdown
# Reflection: <Mission Name> — <Date>

## What Changed
- <Summary of files/modules added, modified, or removed>

## What Worked Well
- <Patterns, tools, or decisions that saved time>

## What Didn't Work
- <Pain points, dead-ends, or wasted effort>

## Suggested Improvements
- <Prompt tweaks, new rules, or process changes for future sessions>
```
